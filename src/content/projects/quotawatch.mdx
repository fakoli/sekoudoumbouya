## Problem

Managing cloud service quotas across multiple providers has become a critical but often overlooked aspect of infrastructure operations. Teams frequently encounter deployment failures when they hit unexpected limits on VPC subnets, Transit Gateway attachments, or PrivateLink endpointsâ€”quotas that aren't always visible until it's too late.

Existing cloud provider consoles only show quotas for individual accounts and services, making it impossible to get a unified view across AWS, GCP, and Azure environments. Documentation is scattered, often outdated, and doesn't provide context about which quotas are adjustable or likely to impact production workloads. I needed a solution that could proactively monitor these limits and provide early warnings before they blocked critical deployments.

The challenge was particularly acute for networking quotas, where limits like CloudWAN connections or Route 53 Resolver endpoints can silently prevent infrastructure expansion. Without a centralized monitoring system, teams resort to manual spreadsheet tracking or discover quota issues only during failed deployments.

## Approach

I built QuotaWatch as a Python CLI tool that creates a unified quota intelligence system across cloud providers. The architecture centers around a local SQLite database that stores quota definitions, current values, usage metrics, and enriched metadata from both APIs and documentation sources.

The core design separates quota data collection into multiple layers: a provider abstraction for cloud API integration, an intelligent documentation parser using Anthropic Claude for context extraction, and an enrichment engine that combines API data with parsed documentation to create comprehensive quota profiles. This layered approach ensures that quota information remains accurate while providing the contextual intelligence needed for effective monitoring.

### Key Design Elements

- **Multi-provider abstraction layer** with pluggable cloud provider clients for AWS Service Quotas API and future GCP/Azure support
- **Intelligent documentation parsing** using Claude to extract quota information from provider documentation and maintain up-to-date context
- **Real-time usage monitoring** that tracks utilization percentages and identifies quotas approaching critical thresholds
- **Configurable watchlist system** for priority-based monitoring of business-critical quotas
- **SQLite-based enrichment engine** that cross-references API data with documentation for comprehensive quota intelligence
- **Comprehensive filtering and querying** with CLI commands for service-based, adjustability, and utilization-based quota discovery

## Outcomes

QuotaWatch enables proactive quota management that prevents deployment failures and supports informed capacity planning decisions. Teams can identify approaching limits before they impact production workloads, while the unified multi-cloud view eliminates the need to check multiple provider consoles.

The intelligent documentation parsing keeps quota context current without manual maintenance, while the enrichment system provides the operational intelligence needed to prioritize quota increases and understand the impact of different limits on infrastructure expansion.
